Twitter Following vs In-Game Performance of NBA Players¶

Introduction¶

Perhaps more so than in any other American professional sports leage, NBA (National Basketball Association) players are very active off the court, often becoming notable celebrities, social figures, and - in many cases - brand ambasadors.

But athletes' presence in popular culture varies wildly from one player to the next. In this tutorial, we attempt to use players' in-game statistics to predict the size of their Twitter following. In addition to just being a potentially interesting correlation to uncover, this model could be used to identify which players have the potential to accrue a larger social impact than the currently have, and thus who could provide a better "bang for your buck" as an endorser.

Now let's dive into the code! We'll start with the imports we need to make the rest of the code work.

Data Collection¶

As is the first step in any good data science project, we need to collect all the data that we will later be performing analysis on. We are going to need gameplay statistics from NBA players (from basketball-reference.com) and also statistics on their Twitter following.

We will start with the gameplay statistics, and to do that we must first compile a list of all the players we are going to investigate. Since Twitter is a relatively new platform compared to the NBA, we will only get data on players who were active during the 2017-18 season, so we do not have to account for the differences caused by being inactive or playing in a previous era. Thus we will get our data from Basketball-Reference's 2018 table.

Basketball-Reference does not have a public API, so to get the data, we will have to scrape their site. Parsing HTML manually can be tough, so we instead use an existing library called Beautiful Soup. Experienced data scientists will likely already be familiar with this library, but if you're a newbie it's a great starting point as it allows you to very easily ingest data you find online. This guide from Dataquest provides more in-depth information than will be in our tutorial.

By manually inspecting the table element we want to ingest, we find that each row is a td element with its data-stat attribute set to player, so we will use Beautiful Soup to parse the page and find all such elements. We will then use a regex to find the the reference to each player's stats page on Basketball Reference.

Those results will be loaded into a Pandas data frame. Pandas is an open source library providing high-performance, easy-to-use data structures and data analysis tools in Python. It allows us to quickly and easily store, aggregate, and query the data that we have collected. If you are interested in learning more about Pandas and its capabilities beyond those used in this tutorial, check out the tutorials they have posted on their website.

Now that we have all the players' internal reference links, we will need to be able to turn one of these links into a set of statistics about the player. In particular, we want the career stats Basketball Reference provides (games, games started, various shooting percentages, and various per-game statistics like points, shots, blocks, etc.) The page also contains the player's salary, which seems like it may help predict popularity, so we will include that in the record as well. Finally, we put the provided reference to the data so we can link it up with the rest of the record later.

In [5]:
def get_player_stats(ref) :
    player_stats_text = requests.get('https://www.basketball-reference.com' + ref).text.replace('<!--', '').replace('-->', '')
    player_stats_soup = bs(player_stats_text, 'html.parser')

    player_stats_html = player_stats_soup.find('table',attrs={"id":"per_game"}).find('tfoot').find('tr')
    player_stats = {x['data-stat']: x.text for x in player_stats_html.find_all('td') if len(x.text) > 0}
    player_stats['salary'] = player_stats_soup.find_all('td', attrs={"data-stat": "salary"})[-2]['csk']
    player_stats['ref'] = ref
    return player_stats

The page also has some information they call 'Bling,' which are basically just awards that the player has receive during their career. These awards include All-Star appearences, NBA Championship wins, and Most Valuable Player wins. These are events that bring a great deal of media attention to a player, so we believe they map impact social media following. As before, we use Beautiful Soup to parse the page and find the 'Bling' elements, then use regular expressions to extract the number of each of these awards they received.

In [6]:
all_star_re = re.compile('(\d+)(x?) All Star')
champ_re = re.compile('(\d+)(x?) NBA Champ')
mvp_re = re.compile('(\d+)(x?) MVP')

def get_player_bling(ref) :
    player_stats_text = requests.get('https://www.basketball-reference.com' + ref).text.replace('<!--', '').replace('-->', '')
    player_stats_soup = bs(player_stats_text, 'html.parser')
    bling_html = player_stats_soup.find(attrs={"id": "bling"})
    if not bling_html:
        return {'ref': ref, 'all_star': 0, 'champ': 0, 'mvp': 0}
    bling_text = bling_html.text
    all_star_match = all_star_re.search(bling_text)
    champ_match = champ_re.search(bling_text)
    mvp_match = mvp_re.search(bling_text)

    player_bling = {'ref': ref}
    player_bling['all_star'] = 0 if not all_star_match else 1 if all_star_match.group(2) == '' else int(all_star_match.group(1))
    player_bling['champ'] = 0 if not champ_match else 1 if champ_match.group(2) == '' else int(champ_match.group(1))
    player_bling['mvp'] = 0 if not mvp_match else 1 if mvp_match.group(2) == '' else int(mvp_match.group(1))
    return player_bling

Finally, we will need to get data related to each of the player's Twitter handles. Given a handle, we will get the number of followers and number of statuses (tweets) associated with that account. We have to deal with a little bit of dirty data at this step, because Basketball Reference's data is not perfect. The two main issues with the data set are that some handles are out of date (player changed handle since being added to table) and others simply have typos.

In the case of a typo or an account which no longer exists, we simply return an empty record. The problem is more difficult when we are pointed to an existing Twitter account, but one that does not actually belong to the player. We will get into how we solve this later in the tutorial.

In [7]:
def get_handle_followers(handle, api) :
    try :
        user = api.get_user(handle)
        return {'handle': handle, 'followers_count': user.followers_count, 'statuses_count': user.statuses_count}
    except :
        return {}

Initial Model¶

Now that we have conducted our necessary EDA, we are ready to start creating a model. To start out, we will create a multiple linear regression model on the features we have deemed potentially important. Thos os way too many features and will make our model hard to understand and interpret, but it provides us with a ceiling for our model's performance. For now, we will train and test on all of our data, but we will NOT do this later when we are testing our finalized model.

In [30]:

linear_reg = LinearRegression().fit(player_data_df[feature_cols], player_data_df['followers_count'])

We will now score our regression model. The score which is returned to us is our R2 coefficient, a statistical measure of how close the true data is to our fitted regression model. More specifically, it is the percentage of the response variable variation that is explained by our regression model. You can read more about the meaning of this score here.

Our score is a 0.86, which is relatively high. This means that we are able to account for most of the variance in data with our model. We will later try to simplify this model without substantially reducing this score.

Conclusion¶

Our model was able to identify a strong correlation between basketball statistics and number of Twitter followers. But, because of how complicated it is to model Twitter followers and how a basketball player's Twitter following may be influenced by off-court behaviors that are not captured in our data, we cannot say that this model can predict Twitter followers with great accuracy. Also, we were able to collect the top 10 features that predict Twitter follower count, which can be used outside the context of basketball to predict which current players are in positions of influence or would be good candidates to pursue endorsement deals.
